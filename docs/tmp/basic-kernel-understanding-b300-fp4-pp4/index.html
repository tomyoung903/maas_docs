Overview

This is a structure-first NVFP4 PP4 trace rendered with the same rank, phase, layer, module-tree, precision, family, and event views as the existing B300 report. It also separates routed FP4, attention FP8, and unquantized BF16 work.

Primary interpretation

Run Contract

The captured region contains one user request only. Server load, JIT compilation, built-in warmup, and health checks finished before profiler collection was enabled.

Online compute path; isolated transfer boundary The image, NVFP4 checkpoint, PP=4 partition, context and batch limits, FP8 KV cache, automatic FP4 runner, TRT-LLM DSA, and FlashInfer TRT-LLM MoE policy match the live prefill workers. Per the experiment contract, this standalone run flushes cache before every request and has no Dynamo PD transfer, Mooncake, or hierarchical cache. Without disaggregation, SGLang requires the decentralized PP scheduler instead of production's centralized scheduler.
  • The checkpoint is NVIDIA GLM-5.2-NVFP4 on four active GPUs; all eight node GPUs were reserved so no foreign workload could interfere.
  • The request uses TP=1, DP=1, PP=4, 16,384 uncached input tokens, and exactly one generated token.
  • The profiled batch logs cuda graph: False; all ten outputs have the same SHA-256.
