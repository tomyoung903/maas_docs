SGLang Parallelism Mental Model: DP, TP, PP, and CP

A concrete map from launch arguments to physical ranks and runtime process groups. The point is to separate outer replicas, pipeline stages, and the per-stage TP island from attention-only and MoE-only subdivisions.

Mental Model

Do not multiply every flag together. SGLang first builds a world from global TP and PP. Then, inside each PP stage's TP group, attention and MoE get their own coordinate systems. Ordinary DP is different: without DP attention, it launches separate model replicas.

Outer launch
Ordinary DP replicas Only when --dp-size is used without DP attention. Each replica launches its own TP × PP world.
PP stages P stages multiply the distributed world and form cross-stage PP groups.
Per PP stage
Global TP island The parent group of size Tglobal. Dense TP defaults here.
Attention-DP view Only with --enable-dp-attention. It reuses the TP ranks instead of launching more GPUs.
Attention-CP view Context-parallel lanes for attention, commonly used for long prefill.
MoE view Separate MoE-DP, EP, and MoE-TP coordinates inside the same TP island.

Core Formulas

World and PP
W = Tglobal × P
rworld = p × Tglobal + rT

The rank inside a model worker is formed from PP rank and local TP rank. This is why PP is outside the TP-island math.

Attention view
Dattn = Darg when DP attention is enabled, otherwise 1
Tattn = Tglobal / (Dattn × Cattn)
rT = ((dattn × Cattn + cattn) × Tattn) + tattn
MoE view
Tmoe = Tglobal / (Dmoe × E)
rT = ((dmoe × E + e) × Tmoe) + tmoe

This is a separate coordinate system from attention. That is the mechanism that keeps MLP/MoE paths from accidentally inheriting attention-DP lanes.

Ordinary DP
GPU budget = Darg × Tglobal × P

This formula is only for ordinary DP when DP attention is disabled. With DP attention, Darg becomes Dattn inside the TP island instead.

Input Args

Arg Symbol Runtime effect Common pitfall
--tp-size Tglobal Parent TP island per PP stage. Attention and MoE derive smaller views from this budget. It is not always the effective attention TP size.
--pp-size P Multiplies the worker world and creates pipeline-stage groups across matching TP ranks. It is outside the attention and MoE subdivisions.
--dp-size Darg Ordinary DP replicas if DP attention is off; attention-DP lanes if DP attention is on. Same flag, two different runtime meanings.
--enable-dp-attention Dattn Reuses the global TP group as attention-DP lanes. It does not multiply physical ranks by DP. Without this flag, --dp-size 8 --tp-size 8 is ordinary DP and wants many more GPUs.
--attn-cp-size Cattn Context-parallel lanes inside attention. Effective attention TP is divided by this value. It must divide the TP island together with attention DP.
--enable-prefill-cp, --cp-strategy policy Turns on prefill CP behavior and selects the split strategy, such as zigzag or interleave. The policy changes the token split pattern, not the group-size formula.
--ep-size E Expert-parallel lanes for MoE layers. EP is MoE-specific. It is not attention TP.
--moe-dp-size Dmoe MoE data-parallel lanes inside the TP island. When it differs from attention CP, current code only allows the special Dmoe = 1 case; current validation also keeps Dmoe > 1 away from PP.

The high-order rule is: PP and ordinary DP can create more model worlds. DP attention, CP, EP, and MoE-DP are views inside the TP island for a specific subsystem.

Rules of Thumb

Question Answer
Does --dp-size multiply GPU count? Yes for ordinary DP. No for DP attention, where it becomes Dattn inside the TP group.
Does CP multiply GPU count? No. Cattn subdivides the TP island, reducing effective attention TP.
Where does PP fit? PP is outside the TP-island formulas. It multiplies the distributed world as W = Tglobal × P.
Why does MLP not get confused by attention DP? Attention, dense TP, and MoE use different process-group views. The same local TP rank can carry attention coordinates and MoE coordinates at the same time.
What must divide cleanly? Tglobal must be divisible by the active attention subdivision and by the active MoE subdivision. Current validation also enforces extra MoE-DP and CP compatibility rules.
