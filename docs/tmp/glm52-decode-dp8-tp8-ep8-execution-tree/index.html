GLM-5.2 standard decode topology

How DP8, TP8, and EP8 share one eight-GPU node

DP8, TP8, and EP8 are not three independent dimensions multiplied into 512 GPUs. In this run, the same physical ranks 0–7 change roles as each decoder layer moves from attention to dense FFN or sparse MoE and back to local decode output.

8× H200 TP=8 / DP=8 / EP=8 / PP=1 DP attention + DP LM head DeepEP low-latency standard decode, no EAGLE

Short Answer

The important correction: it would be inaccurate to label 2.1.2 as DP and 2.1.3 as TP. Under this topology, both are DP-local attention work because the effective attention TP group has size one. TP8 first appears at the dense FFN boundary; EP8 first appears at sparse-MoE dispatch.
Physical world
8 = TP8 × PP1
The process world contains exactly eight GPU ranks.
Attention group
attn_TP = 8 / 8 / 1 = 1
Each DP rank has a complete attention copy and owns its local token rows.
MoE expert tensor group
MoE_TP = 8 / 8 / 1 = 1
An expert is owned whole by one EP rank; its matrices are not split across eight ranks.
Expert group
EP group = ranks 0..7
Because EP equals TP, SGLang reuses the same eight-rank group.

DP describes which rank owns each request/token row for attention and output. TP describes how dense FFN weight matrices are split. EP describes which ranks own the MoE experts and where selected token rows must travel. These are data-layout and ownership roles, not three separate pools of hardware.

Layer Handoffs

Every decoder layer starts with local attention. The feed-forward branch then selects one of two layouts. Layers 0–2 take the dense TP path; layers 3–77 take the sparse EP path.

2.1 through 2.2 DP-local attention The token-owning rank runs the complete norm, Q/K/V, DSA, MLA, output projection, residual, and post-attention norm.
Dense layers 0–2 TP8 FFN Gather DP rows, run sharded gate/up and down matrices on all ranks, then reduce-scatter.
Sparse layers 3–77 EP8 MoE Route locally, dispatch token copies to expert owners, run whole local experts, then combine back.
2.4, then next layer Return to DP-local The local residual path resumes. After layer 77, final norm and LM head remain local.
Dense branch: DP rows → TP8 → DP rows

The all-gather is a data-layout bridge, not a model layer. Each rank receives the global token rows, computes its weight shard, and the down-projection partials are summed while the rows are scattered back to their DP owners.

Sparse branch: DP rows → EP8 → DP rows

No full-row all-gather is needed. Each source DP rank computes routing for its own rows. DeepEP dispatches selected copies directly to expert owners and combines their weighted outputs back to the source rank.

One Real Decode Token

Assume one real token is assigned to DP rank 3. The rank number is only an example; the scheduler can choose any DP rank.

Local attentionRank 3 owns the token, KV cache, DSA selection, and complete attention path for all 78 layers.
Dense layersFor layers 0–2, the row is gathered to all eight ranks. Each computes a dense weight shard.
Sparse layersFor layers 3–77, rank 3 routes locally. Token copies travel only to the selected expert owners.
Return to ownerReduce-scatter or DeepEP combine reconstructs the output and returns it to rank 3 after each FFN.
Local logitsRank 3 runs final norm and its local full-vocabulary LM head, then emits the next token.
Logical ownership is not the same as profiler visibility. Only one DP rank owns the real token in this example, but full CUDA-graph capture, padding, and required collectives can still produce kernels on all eight GPUs. A kernel on an otherwise idle rank does not imply that rank owns another real request.
