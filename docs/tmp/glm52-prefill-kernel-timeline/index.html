Overview

This page is the published, MaaS Docs formatted version of the GLM-5.2 prefill-shaped profiling run captured on gpu-worker-204. The run launched a standalone PP=8 SGLang server under host Nsight Systems, exercised it with a max_tokens=1 stress-test shape, and rendered the CUDA kernel activity into readable timeline charts and tables.

What Was Run

One temporary Kubernetes pod, pinned to gpu-worker-204, used the host Nsight binary with SQLite export enabled. The server used GLM-5.2 FP8, --pp-size 8, --tp-size 1, --dp-size 1, and chunked prefill settings matching the prefill-only experiment shape.

What Was Measured

The profile contains 78,946 visible CUDA kernel launches across 116 distinct kernel-name groups. Aggregate kernel time is 119.01 s, summed across 8 GPUs, over a 117.75 s profiler kernel span.

How To Read It

The first two charts show kernel time density and kernel launch density per one-second bin. Scheduler charts are based on SGLang log lines, not CUPTI events. Kernel totals are grouped by operation family first, then the full per-kernel table gives the raw symbol evidence.

Main Caveat

This is a standalone prefill-shaped measurement, not a full disaggregated prefill/decode KV-transfer test. The SGLang log has 72 prefill rows and 0 decode rows. The Locust window produced server-side prefill work but no completed client rows under profiler overhead.

Published artifacts: timeline_data.json and kernel_summary.csv.

Known limitations

  • This was a standalone prefill-shaped run on gpu-worker-204, not a full disaggregated PD bootstrap/KV-transfer test.
  • The Locust client run used max_tokens=1 and produced server-side prefill work, but no completed rows in its 30 second client window under Nsight overhead.
  • A direct /generate check with max_new_tokens=1 completed and is recorded in the SGLang Finish log.
  • SGLang logs in this capture show 72 prefill batch rows and 0 Decode batch rows.
  • CUPTI kernel rows have no graphId/graphNodeId values, so the CUDA graph panel is reported as unavailable/zero.
  • Aggregate GPU time is summed across 8 devices and is not wall-clock time.
