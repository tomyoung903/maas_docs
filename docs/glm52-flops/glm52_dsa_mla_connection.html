Absorbed Key And Value Math

The latent cache is not the same as ordinary per-head K and V tensors. For scoring, each head has a learned content-key up-projection. For values, the attention kernel can first accumulate the shared latent vector and apply the value up-projection after the weighted sum.

DSA Selection Math

DSA adds a learned indexer before sparse attention. The indexer is a separate relevance scorer: it looks at the current query token and the cached index keys for causal candidate tokens up to and including the current position, then returns token positions to keep. It does not produce the final attention probabilities.

Core Scaling Consequence

Without sparse selection, a new query's final attention sum scans every allowed key. With DSA, the final MLA attention sum is capped by K once the context is longer than K. The selector still scores candidates over the causal history before top-k; that selector surface is the long-prefill system bottleneck discussed in the Prefill CP page.
