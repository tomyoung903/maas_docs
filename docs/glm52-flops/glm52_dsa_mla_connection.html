GLM-5.2 DSA + MLA Connection

MLA defines the compressed attention representation. DSA decides which context keys each query should scan. DSA prefill CP decides which GPU rank owns which query tokens during long prefill.

The Three Pieces

NameWhat it isWhat it changes
MLAMulti-head Latent Attention. Query and KV are projected through low-rank latent spaces: q_lora=2,048, kv_lora=512, and qk_rope=64.It reduces the KV cache shape and lets the attention score/value math use absorbed projection matrices.
DSADeepSeek Sparse Attention. An indexer predicts top-k useful context keys for each query; GLM-5.2 uses index_topk=2,048.It changes the long-context scan from all prior keys to selected keys after the dense threshold.
DSA prefill CPRuntime context parallelism for DSA prefill. It splits query tokens across attention-CP ranks while keeping attention TP at 1 in the H200 TP8 setup.It reduces per-rank long-prefill attention work and memory pressure along the token axis.

GLM-5.2 H200 CP Shape

tp_size = 8
dp_size = 1
attn_cp_size = tp_size / dp_size = 8
attn_tp_size = tp_size / dp_size / attn_cp_size = 1

rank r owns query tokens where token_id % 8 == r

for each local token t:
  q_latent_t = q_a(h_t)
  q_t        = q_b(norm(q_latent_t))   # full 64 heads, not 8 heads
  topk_t     = DSA_indexer(h_t, q_latent_t)
  out_t      = sparse_MLA(q_t, selected_KV[topk_t])
The important part: under DSA prefill CP, attn_tp_size=1. Each rank computes full-head Q for about one eighth of the query tokens. It is not computing one eighth of the heads for all query tokens.

TP vs CP For Query Projection

LayoutRank seesQ shape per rankMeaning
Classic attention TP8all query tokens[T, 8, 256]Output/head dimension is sharded across TP ranks.
DSA prefill CP8about one eighth of query tokens[T/8, 64, 256]Sequence/query-token dimension is sharded across CP ranks.

For the main q_b_proj GEMM, CP is mostly a different sharding axis: T * Q_out/8 versus T/8 * Q_out. The larger win is not that Q projection has fewer total model FLOPs. The larger win is that long-prefill attention and DSA selected-key scanning are distributed across ranks by query token.

What Is Not Happening

MisreadCorrect reading
tp=8 means Q is still head-sharded during DSA CP.No. In this setup attn_cp_size=8 consumes the TP group for attention CP, so effective attn_tp_size=1.
DSA is the same thing as MLA.No. MLA is the compressed attention representation. DSA is the sparse key-selection mechanism sitting on top of that attention path.
CP makes Q projection algorithmically cheaper.Not really for q_b_proj; it changes the sharding axis. The main prefill benefit is attention scan parallelism and lower per-rank pressure.
The MTP layer is always active.No. The checkpoint has one nextn/MTP layer, but a normal non-speculative forward does not use it. SGLang uses it when EAGLE speculative decoding is enabled.

Practical Reading For GLM-5.2

When someone says GLM-5.2 uses MLA + DSA, read it as: the model stores and computes attention through DeepSeek-style latent KV, then uses a DSA indexer to choose which latent KV entries each query should attend to at long context. When someone says the H200 setup uses DSA prefill CP, read it as: the long prefill query tokens are split across the eight GPUs, and each GPU runs full-head MLA/DSA attention for its local token slice.
