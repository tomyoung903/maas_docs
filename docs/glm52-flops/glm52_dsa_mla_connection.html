MLA Projection Math

MLA first maps the hidden state into a query latent and a compact KV cache entry. The cache stores one shared latent vector plus one RoPE key vector per token and layer.

query latent:
  z_t^q = W_DQ h_t                         in R^2048
  q_t   = reshape(W_UQ RMSNorm(z_t^q))      in R^64 x R^256

cached KV payload:
  [c_t^kv, k_t^R] = W_DKV h_t
  c_t^kv in R^512
  k_t^R  in R^64

cache scalars per token per layer:
  r_kv + d_r = 512 + 64 = 576
A materialized per-head KV would have H * (d_q + d_v) = 64 * (256 + 256) = 32,768 scalars per token per layer. MLA stores 576, about 56.9x fewer scalars before dtype bytes and layer count.

Absorbed Key And Value Math

The latent cache is not the same as ordinary per-head K and V tensors. The model can recover the content score and value contribution from the latent vector through learned up-projections, often algebraically absorbed into the attention kernels.

split the query head:
  q_{t,a} = [q^C_{t,a}, q^R_{t,a}]
  q^C_{t,a} in R^192
  q^R_{t,a} in R^64

for a previous token i and head a:
  k^C_{i,a} = W_UK,a c_i^kv              in R^192
  v_{i,a}   = W_UV,a c_i^kv              in R^256

attention score:
  s_{t,i,a} =
    dot(q^C_{t,a}, k^C_{i,a})
    + dot(RoPE(q^R_{t,a}, t), k^R_i)
    ------------------------------------
              sqrt(d_q)

DSA Selection Math

DSA adds a learned indexer before sparse attention. The indexer estimates which past token positions matter for the current query, then attention uses only that selected set.

causal candidate set:
  C_t = {i | 0 <= i <= t}

indexer logits:
  u_{t,i} = Indexer(h_t, z_t^q, cached_index_key_i)

selected positions:
  I_t = TopK_i(u_{t,i}, K=2048) over i in C_t

short context:
  if |C_t| <= K, then I_t = C_t

Sparse MLA Attention

After DSA chooses I_t, the softmax is normal attention math, but its denominator and value sum range over selected positions only.

per head a:
  alpha_{t,i,a} = softmax over i in I_t of s_{t,i,a}

  o_{t,a} = sum over i in I_t:
              alpha_{t,i,a} * v_{i,a}

combine heads:
  o_t = concat_a(o_{t,a})                in R^16384
  y_t = W_O o_t                          in R^6144

Core Scaling Consequence

Without sparse selection, a new query scans every previous key. With DSA, the scan term is capped by K once the context is longer than K. This is the fundamental long-context change.

dense selected positions:
  |I_t| = t + 1

DSA selected positions:
  |I_t| = min(t + 1, 2048)

rough all-head MLA scan per selected key:
  H * 2 * (2 * r_kv + d_r)
  = 64 * 2 * (2 * 512 + 64)
  = 139,264 FLOPs / selected key / layer

rough DSA scan cap per layer:
  139,264 * 2048
  = 285,212,672 FLOPs / token / layer

rough DSA scan cap across 78 layers:
  285,212,672 * 78
  = 22.247 GFLOP / token
This page counts only the model-side attention scan shape. The DSA indexer, top-k operation, cache movement, and kernel overhead are additional system costs; they do not change the mathematical fact that the attention sum is over I_t instead of all previous positions.
