MLA Projection Math

MLA first maps the hidden state into a query latent and a compact KV cache entry. The cache stores one shared latent vector plus one RoPE key vector per token and layer.

query latent:
  z_t^q = W_DQ h_t                         in R^2048
  q_t   = reshape(W_UQ RMSNorm(z_t^q))      in R^64 x R^256

cached KV payload:
  [c_t^kv, k_t^R] = W_DKV h_t
  c_t^kv in R^512
  k_t^R  in R^64

cache scalars per token per layer:
  r_kv + d_r = 512 + 64 = 576
A materialized per-head KV would have H * (d_q + d_v) = 64 * (256 + 256) = 32,768 scalars per token per layer. MLA stores 576, about 56.9x fewer scalars before dtype bytes and layer count.

Absorbed Key And Value Math

The latent cache is not the same as ordinary per-head K and V tensors. For scoring, each head has a learned content-key up-projection. For values, the attention kernel can first accumulate the shared latent vector and apply the value up-projection after the weighted sum.

split the query head:
  q_{t,a} = [q^C_{t,a}, q^R_{t,a}]
  q^C_{t,a} in R^192
  q^R_{t,a} in R^64

for a previous token i and head a:
  k^C_{i,a} = W_UK,a c_i^kv              in R^192
  c_i^kv is also the latent value input       in R^512

attention score:
  s_{t,i,a} =
    (dot(q^C_{t,a}, k^C_{i,a})
     + dot(RoPE(q^R_{t,a}, t), k^R_i)) / sqrt(d_q)

DSA Selection Math

DSA adds a learned indexer before sparse attention. The indexer is a separate relevance scorer: it looks at the current query token and the cached index keys for previous tokens, then returns token positions to keep. It does not produce the final attention probabilities.

For GLM-5.2, the indexer lives in its own learned space with 32 index heads, index head dimension 128, and K=2048 selected positions. Its job is only to choose candidates; the real MLA score s_{t,i,a} is computed later for those candidates.

causal candidate set:
  C_t = {i | 0 <= i <= t}                         # L_t integer positions

conceptual indexer query:
  z_norm_t^q = RMSNorm(z_t^q)                        # [2048]
  x_t = reshape(z_norm_t^q @ W_idx_Q)                # [32, 128]
  x_t[:, :64] = RoPE(x_t[:, :64], position=t)
  x_t = Hadamard(x_t)                                # [32, 128]
  # SGLang implementation name: Indexer.wq_b

per-query index-head gates:
  g_t = (h_t @ W_idx_G) / sqrt(32)                   # [32]
  # SGLang implementation name: Indexer.weights_proj

cached index key for a previous token:
  kidx_i = LayerNorm(h_i @ W_idx_K)                  # [128]
  kidx_i[:64] = RoPE(kidx_i[:64], position=i)
  kidx_i = Hadamard(kidx_i)                          # [128]
  # SGLang implementation name: Indexer.wk, cached as cached_index_key_i
  Kidx_t = stack_i(kidx_i for i in C_t)              # [L_t, 128]

raw index-head scores:
  m_{t,i,b} = relu(dot(x_t[b], kidx_i))            # scalar
  M_t = score_heads(x_t, Kidx_t)                     # [L_t, 32]

relevance logits:
  u_{t,i} = sum_b g_t[b] * m_{t,i,b}             # scalar
  u_t = [u_{t,i} for i in C_t]                     # [L_t]

selected positions:
  I_t = TopK_i(u_t, K=2048)            # logical int positions, [min(K, L_t)]
  TopK_t = pad_with_minus_one(I_t, K=2048)  # implementation tensor, [2048]

short context:
  if |C_t| <= K, then I_t = C_t
The equations above are the model-math view. SGLang quantizes the indexer query/key path for the fast kernel and folds FP8 scales plus the head_dim^-0.5 score scale into the implementation weights.

Read I_t as an integer position list, not as attention weights. DSA first chooses positions with the indexer; the next section computes the actual softmax over only those positions.

Sparse MLA Attention

After DSA chooses I_t, the softmax is normal attention math, but its denominator and value sum range over selected positions only.

per head a:
  alpha_{t,i,a} = softmax over i in I_t of s_{t,i,a}

latent value accumulation:
  z^attn_{t,a} = sum over i in I_t:
                   alpha_{t,i,a} * c_i^kv          # [512]

per-head value up-projection:
  o_{t,a} = W_UV,a z^attn_{t,a}                   # [256]

combine heads:
  o_t = concat_a(o_{t,a})                in R^16384
  y_t = W_O o_t                          in R^6144

Core Scaling Consequence

Without sparse selection, a new query scans every previous key. With DSA, the scan term is capped by K once the context is longer than K. This is the fundamental long-context change.

dense selected positions:
  |I_t| = t + 1

DSA selected positions:
  |I_t| = min(t + 1, 2048)

model-math MLA scan per selected key:
  H * 2 * (2 * r_kv + d_r)
  = 64 * 2 * (2 * 512 + 64)
  = 139,264 FLOPs / selected key / layer

model-math DSA selected-key scan cap per layer:
  139,264 * 2048
  = 285,212,672 FLOPs / token / layer

model-math DSA selected-key scan cap across 78 layers:
  285,212,672 * 78
  = 22.247 GFLOP / token
This page counts only the model-side attention scan shape. The DSA indexer, top-k operation, cache movement, and kernel overhead are additional system costs; they do not change the mathematical fact that the attention sum is over I_t instead of all previous positions.
