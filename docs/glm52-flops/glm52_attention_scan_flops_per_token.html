GLM-5.2 Attention Scan FLOPs

Attention scan work grows with the number of context keys touched by each generated token. GLM-5.2's DSA path limits the selected-key scan to top-2048 keys after the context exceeds that threshold.

Per Context-Pair Formula

local_heads = heads / TP = 64 / 8 = 8

per_layer_pair =
  local_heads * 2 * (2 * kv_lora + qk_rope)
= 8 * 2 * (2 * 512 + 64)
= 17,408 FLOPs/context-pair/layer/GPU

all_layers_pair =
  17,408 * 78
= 1,357,824 FLOPs/context-pair/GPU

The DSA selected-key column is not total attention cost. It is the score/value scan over selected keys. The indexer, top-k selection, metadata handling, memory traffic, and backend-specific kernels still need profiling or separate accounting.

Reading This Correctly

Below 2,048 context tokens, the dense and selected-key scan counts match. Above 2,048, the selected-key scan term stays flat while the dense full-context counter keeps growing linearly. This is why GLM-5.2 long-context profiling should not use a naive dense-attention scan model as the only MFU denominator.
