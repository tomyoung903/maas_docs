Scope

Every target layer computes its own MLA attention over the selected positions. Only 21 layers run a learned DSA indexer to refresh those integer positions; 57 layers reuse the latest list. Reuse avoids a new selector calculation, but it does not skip that layer's MLA score, softmax, latent-value accumulation, or output projection.

This page therefore multiplies the selected-key MLA pair cost by 78, not 21. Conversely, DSA candidate scoring belongs only to the 21 refresh layers and is outside this scan constant. The 100K prefill derivation places both terms at their correct layer scope.

The DSA selected-key column is not total attention cost. It is the score/value scan over selected keys. The indexer, top-k selection, metadata handling, memory traffic, and backend-specific kernels still need profiling or separate accounting.

Reading This Correctly

Below 2,048 context tokens, the dense and selected-key scan counts match. Above 2,048, the selected-key scan term stays flat while the dense full-context counter keeps growing linearly. This is why GLM-5.2 long-context profiling should not use a naive dense-attention scan model as the only MFU denominator.
