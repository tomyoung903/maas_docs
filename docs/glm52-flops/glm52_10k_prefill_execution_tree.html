Expand The 78 Layers

Only three decoder patterns are needed to reconstruct the full stack. Every layer still executes its own MLA selected-key attention; the distinction is whether the layer refreshes DSA TopK positions and whether its feed-forward block is dense or sparse.

Reconstruction rule: execute layers 0 through 77 in numerical order. At every layer, follow the common attention path, choose refresh or reuse according to the table, then choose the dense or sparse feed-forward branch.

MFU Handoff

The tree supplies operation order, full tensor dimensions, and execution multiplicity. A prefill FLOP model should apply an operation-specific formula to each leaf and then sum the leaves; parameter count alone is not a FLOP count.

Execution termLogical multiplicity and shapePrefill FLOP treatment
EmbeddingLookup count and [10000, 6144] outputMemory lookup; do not charge the full vocabulary matrix as a GEMM
Linear projections10,000 token rows and the full matrix dimensions shown in the treeFor a matrix product, use 2 × m × k × n FLOPs
DSA selector21 refresh layers with [10000, 2048] selected-position outputCount selector projections and causal candidate scoring only in refresh layers; include TopK separately if the MFU convention requires it
MLA selected-key scan64 heads; selected positions capped at 2,048 in all 78 layersCount both score dot products and latent-value accumulation using the causal selected-pair sum
Dense MLP3 layers with intermediate width 12,288Count merged gate/up and down matrix multiplications, then state separately whether elementwise SiLU is included
Sparse MoE75 layers; router [10000, 256]; active routes [10000, 9]Charge 8 routed experts plus 1 shared expert per token, not all 257 logical experts
Output headOne projected hidden row, then [1, 154880] full-vocabulary logitsCharge one vocabulary projection row, not all 10K hidden rows
Norm, RoPE, softmax, routingShapes and repetition are explicit tree leavesKeep as named non-GEMM terms; include or exclude explicitly instead of hiding them inside a parameter multiplier
