Collapsed Execution Tree

Read from top to bottom. Indentation means ownership: the decoder stack owns a repeated decoder layer, the layer owns an attention sublayer and feed-forward sublayer, and each sublayer owns its calculation-ordered operations. Parent FLOP cells are subtotals; expanded leaf cells reconcile those subtotals without counting parent and child values twice.

Expanded arithmetic convention: one scalar add, multiply, divide, reciprocal square root, or exponential counts as one operation; a multiply-add remains two FLOPs. This makes norms, RoPE, softmax, SiLU, routing arithmetic, residuals, and the output head part of the total. TopK comparisons, gathers, permutations, ID writes, and communication are displayed as auxiliary work because they are not floating-point operations.
Precision display: the main FLOP value remains the additive raw-logical ledger. At verified nodes, the inline H200 precision view converts only Tensor-Core work to FP8-equivalent units and keeps FP32 scalar/SFU work separate.

Expand The 78 Layers

Only three decoder patterns are needed to reconstruct the full stack. Every layer still executes its own MLA selected-key attention; the distinction is whether the layer refreshes DSA TopK positions and whether its feed-forward block is dense or sparse.
