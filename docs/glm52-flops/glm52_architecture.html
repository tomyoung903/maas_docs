Why Both Query-B And KV-B Count

q_b_proj does not directly emit the 512-dimensional absorbed MLA query. It first expands the 2,048-dimensional query latent into eight local heads of 192 content coordinates plus 64 RoPE coordinates. The key half of kv_b_proj is then absorbed into the content query, while its value half expands the latent attention result after the scan.

SGLang splits the learned kv_b_proj weight into w_kc and w_vc during post-load processing. The first BMM maps the 192-D content query into the 512-D cached-latent space; the second maps the 512-D latent attention result into the 256-D value space. See the absorbed MLA derivation.

DSA / NSA Sparse Attention

All 78 target-model layers run selected-key MLA attention. GLM-5.2 does not recompute the learned DSA selector in every layer: 21 layers refresh the TopK positions, while the other 57 layers reuse the positions carried from the most recent refresh layer and apply their own layer-local MLA projections and KV cache.

The first three layers each refresh. Layer 2 supplies layers 3-5; layer 6 supplies layers 7-9; the four-layer cadence continues through layer 74 supplying layers 75-77. The selector uses 32 index heads of width 128.
The parameter table's 21-module count is therefore correct: only refresh layers carry learned indexer weights in the target checkpoint. The separate MTP layer has its own layer-78 indexer weights and is outside this target-model count.

Speculative Decoding

The checkpoint has num_nextn_predict_layers = 1. A plain non-speculative forward does not use it. In our SGLang serving setup, --speculative-algorithm EAGLE loads the same checkpoint as the draft model and remaps the nextn layer into the draft path.
