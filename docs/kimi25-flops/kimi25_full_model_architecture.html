This is the repeated unit that creates almost all per-token compute. The variable names below are intentionally the same as the two FLOPs pages.

The residual adds and RMSNorms are drawn because they are architecturally important, but they are intentionally excluded from linear_flops_per_token.

Block input hidden[H] = [T,7168] H = 7168. Residual stream entering one of L = 61 text blocks.
MLA projections qkv_a -> q_lora + kv_lora + qk_rope-key branch q_lora=1536, kv_lora=512, qk_rope=64.
MLA attention scan Content QK + RoPE QK + AV latent Uses all heads=64. The cached content vector is normalized k_nope/c_j[512]. Pair cost is attention_scan_flops_per_pair.
MLP or MoE dense layer 0, MoE layers 1-60 dense_intermediate=18432, moe_intermediate=2048, active_experts=9.
Block output hidden[H] = [T,7168] Returns to the residual stream and feeds the next block.

5. Zoom-In: MoE Variables

The MoE picture should show sparsity clearly: the router scores all routed experts, but only a small active set runs for each token.

Router gate hidden[H] -> routed_experts routed_experts=384. Router score matmul is included in the linear constant.
Top-k routed topk = 8 Only 8 routed experts are selected by the router for the token.
Shared expert shared_experts = 1 Always-on shared path, not router-selected. In the FLOP accounting, active_experts=9.
Expert MLP shape H -> moe_intermediate -> H moe_intermediate=2048, SwiGLU gate/up/down projections.
Not all 384 run dense-MoE would be wrong Counting every expert would wildly overstate active inference FLOPs.
inactive routed expert tile, representing the 384-expert pool 8 router-selected routed experts 1 always-on shared expert 
active_moe_layer =
  6 * H * moe_intermediate * active_experts
  = 6 * 7168 * 2048 * 9
  = 792,723,456 FLOPs/token

6. Worked Serving Walkthrough: Prefill Then Decode

This section connects the architecture picture to serving mechanics. The same block is used in prefill and decode, but the attention-pair count changes because decode reads the accumulated KV cache one new token at a time.

Prefill token uncached prompt token Pays linear_flops_per_token and writes normalized k_nope/c_j[512] plus RoPE key into the cache.
Prefill scan causal mask Each prompt query attends only to earlier positions. Pair count is triangular over the uncached span plus cached-prefix interactions.
Decode token one generated step Pays one linear-token cost for the new token.
KV cache read query x prior context Pays attention_scan_flops_per_pair for each prior context position read by each layer.
Logits + sampling lm_head -> next token Drawn as part of the loop, but excluded from the two constants.
