1. What Is A Context Pair?

A context pair means one query token looking at one cached context position, across all 64 heads on the single giant GPU. The total number of pairs comes from the request shape: prompt length, output length, and cached prefix behavior.

attention_scan_FLOPs =
  attention_pair_count
  * attention_scan_flops_per_pair

Cached prefix tokens are not charged full linear work again, but they can still be keys/values in context. That is why cached tokens can still create attention pairs.

2. Absorbed MLA Scan Shape

Kimi uses absorbed MLA. Instead of materializing full per-head K/V tensors for the cache, the scan works against latent cache state and a small RoPE key.

Symbol Value Meaning inside scan
heads64all attention heads on the single giant GPU
kv_lora512latent content/value cache dimension
qk_rope64position/RoPE key dimension
layers61number of text transformer layers
qk_nope128does not appear in scan; it is absorbed into W_KC before scan
v_head128does not appear in scan; latent output is projected to v_head after scan via W_VC

3. Per-Layer Pair FLOPs

For one query-context pair, each head performs three scan operations: content score, position score, and latent value accumulation.

Scan Part Per Head Formula All 64 Heads FLOPs
Content QK score 2 * kv_lora = 2 * 512 64 * 2 * 512 65,536
RoPE QK score 2 * qk_rope = 2 * 64 64 * 2 * 64 8,192
AV latent accumulation 2 * kv_lora = 2 * 512 64 * 2 * 512 65,536
Total per layer 2 * (512 + 64 + 512) 64 * 2 * 1088 139,264 
attention_scan_flops_per_pair_per_layer =
  2 * heads * (kv_lora + qk_rope + kv_lora)

= 2 * 64 * (512 + 64 + 512)
= 139,264

4. Across All Layers

attention_scan_flops_per_pair =
  layers * attention_scan_flops_per_pair_per_layer

= 61 * 139,264
= 8,495,104 FLOPs/context-pair

Equivalent compact form: 61 * 2 * 64 * (2 * 512 + 64) = 8,495,104.

5. Why qk_nope And v_head Do Not Appear

Question Answer
Why not qk_nope=128? The no-position query is multiplied by W_KC before the scan, turning it into a 512-dim latent content query. That projection is charged in the linear-token constant, not the pair-scan constant.
Why not v_head=128? The scan accumulates latent kv_lora=512 values. The later W_VC projection maps latent output to v_head=128, and is charged in the linear-token constant.

6. What This Constant Does Not Include

Excluded Reason
qkv_a, q_b, W_KC, W_VC, o_projThese are per-forward-token linear work and belong in the 61,023,322,112 constant.
Softmax, scale, mask, exp, reductionsUseful runtime work, but excluded from this matmul-style theoretical model-FLOP constant.
RMSNorm, RoPE elementwise rotationNon-matmul elementwise operations excluded for consistency with the linear constant.
KV cache bytesMemory traffic is a separate bandwidth model, not FLOPs.
