Kimi K2.5 100K Prefill Execution Tree

A calculation-ordered map for one uncached 100,000-token prefill under the same Kimi FLOPs convention used by this pack: major transformer-block matmuls only, with linear-token work separated from causal attention-pair scan work.

prefill input100,000 tokensone uncached prompt
causal pairs5,000,050,000N * (N + 1) / 2
linear-token FLOPs6.102 PFLOP61,023,322,112 FLOPs/token
attention-scan FLOPs42.48 PFLOP8,495,104 FLOPs/pair
model total48.58 PFLOPlinear + scan
Branches Linear-token matmuls Causal MLA scan Dense/ sparse feed-forward Excluded runtime work
Accounting boundary: this page intentionally follows the existing Kimi K2.5 pack. It excludes RMSNorm, RoPE elementwise rotation, scale/softmax scalar math, TopK comparisons, dispatch movement, communication, quantization bookkeeping, lm_head, and sampling. The GLM page uses a broader expanded ledger; this Kimi tree keeps the already-published Kimi constants consistent.

MFU Handoff

The serving-model formula represented by the tree is:

Kimi_model_FLOPs =
  forward_token_count * 61,023,322,112
  + attention_pair_count * 8,495,104

For one uncached 100K prefill:

forward_token_count = 100,000
attention_pair_count = 100,000 * (100,000 + 1) / 2 = 5,000,050,000
Kimi_model_FLOPs = 48,578,276,966,400,000 = 48.58 PFLOP
Audit warning: recompute this tree if the runtime switches to DSA/NSA sparse attention, changes the MLA absorption boundary, includes lm_head or softmax in MFU, or applies model-parallel accounting instead of the single logical-model baseline.
